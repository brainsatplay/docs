## The Repository
**docs** aggregates documentation for the [Brains@Play Framework](https://github.com/brainsatplay/brainsatplay/README.md). This is served at [docs.brainsatplay.com](https://docs.brainsatplay.com)

### Contents
#### [Guides](./old/docs/index.md)
A comprehensive introduction to the Brains@Play ecosystem.

#### [Resources](./old/docs/resources/index.md)
This is a collection of open/free source biomedical hardware and software development and learning infrastructure which is pushing us towards a future of low-cost, community-driven healthcare technology.

#### [Reference](./old/docs/guides/index.md)
An autogenerated reference for the [brainsatplay] library.

### Repo Contents
#### [src](./src)
**docs** autogenerates documentation for Brains@Play applications as they recompose the web. This allows developers to generate thorough documentation automatically!

> **Note**: This is still just a mockup of a docs generation API. It's not gonna really work how you want (yet)!

Uses https://showdownjs.com/ to generate HTML from .md files.

##### Roadmap
- Check out [ESDoc](https://esdoc.org/)
- Aggregate documentation from across all of our repositories into one distribution. 
    - [x] Implement link crawling, so you download remote links based on a specific pattern (e.g. look for "github.com/brainsatplay")
    - [ ] Make relative links to link between transferred content!
    - [ ] Write files directly with new links...
-  Editor with active plug-in + generated explanation from WASL file (see [htil](https://github.com/brainsatplay/htil)).

#### [old](./old)
The old documentation site created in Docusaurus
